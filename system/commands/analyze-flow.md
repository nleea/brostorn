# Analyze Flow

## Goal

Understand an end-to-end code path before making changes.

## Workflow

1. Use `get_context_pack(...)` for the target task or module.
2. Read the distilled note first, then the top supporting notes.
3. Trace entrypoints, service boundaries, data movement, and side effects.
4. Identify the main files, integration points, and hidden dependencies.
5. Summarize the current flow in concise operational language.
6. Record important gaps or ambiguities in a session summary if the analysis will inform later work.

## Output Checklist

- Entrypoints identified
- Main dependencies listed
- Risks or unknowns called out
