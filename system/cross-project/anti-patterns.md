# Anti-Patterns

## Anti-Pattern: Mixing project-specific bugs into the global system

### Problem
The global brain becomes noisy and retrieval quality drops.

### Preferred Approach
Store the detailed bug in the project docs.
Store only the reusable lesson in cross-project memory.

## Anti-Pattern: Loading too much context

### Problem
Token usage increases and the agent loses focus.

### Preferred Approach
Start with distilled context and load only necessary supporting notes.
