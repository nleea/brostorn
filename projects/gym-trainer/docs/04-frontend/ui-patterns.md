# UI Patterns

## Estructura de UI
- Layouts por rol.
- Componentes especializados por feature.
- Stores como fuente de verdad para estado compartido.

## Patrones visibles
- Cards de resumen.
- Modales para evidencias y feedback.
- Gráficas para progreso, volumen y adherencia.
- Formularios para planes y métricas.

## Observaciones
- Hay mezcla de componentes nuevos y legacy.
- Persisten nombres inconsistentes en algunas vistas y repos.
