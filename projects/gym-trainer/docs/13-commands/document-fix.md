Use this workflow after a bug fix or important behavior correction has been implemented.  
  
## Goal  
Create durable engineering memory for future debugging and maintenance.  
  
## Workflow  
  
1. Identify the bug or behavior that was fixed.  
2. Gather:  
- symptom  
- root cause  
- solution applied  
- files modified  
- validation performed  
- follow-ups  
3. Save or update a debugging note in the vault.  
4. If the fix clarifies or changes expected system behavior, update the corresponding architecture or module note.  
5. If the fix reveals a reusable pattern, propose an update to skills or distilled context.  
  
## Rules  
- Do not leave bug fixes undocumented.  
- Keep notes concise but specific.  
- Prefer clear operational language over vague summaries.  
- Record root cause, not just the final code change.  
  
## Output Expectations  
Your response should include:  
- debugging note path  
- summary of what was documented  
- additional notes updated, if any
